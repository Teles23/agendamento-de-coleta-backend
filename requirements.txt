Requisitos Funcionais e Regras de Negócio – Sistema de Coletas Recicláveis:
1. 
Introdução
:
Este documento descreve os requisitos funcionais e regras de negócio para o sistema de agendamento de coletas de recicláveis. O objetivo é permitir que cidadãos agendem coletas de forma prática e que o sistema organize e proteja o acesso às informações, garantindo segurança, rastreabilidade e usabilidade.
2. 
Visão Geral do Fluxo
O cidadão preenche um formulário com suas informações e solicita um agendamento de coleta.
O sistema valida as informações e registra o agendamento como “Pendente”.
Usuários autenticados podem acessar a listagem de agendamentos, com filtros.
O sistema permite que usuários autenticados atualizem o status da coleta.
O cidadão recebe um número de protocolo ao final do processo.
3. 
Requisitos Funcionais (RF) e Regras de Negócio (RN)
RF001: Solicitar Agendamento de Coleta
O sistema deve permitir que qualquer cidadão preencha um formulário para agendar a coleta.
RN001.1: Os dados obrigatórios do formulário são:
Nome completo
Endereço (rua, número, bairro, cidade)
Tipo de material reciclável (seleção múltipla)
Data sugerida para a coleta
Telefone de contato
E-mail (opcional)
RN001.2: A data sugerida deve ser pelo menos 2 dias úteis após a data atual.
RN001.3: Os tipos de materiais disponíveis devem estar previamente definidos no sistema (ex: Papel, Plástico, Vidro, Metal, Eletrônicos).
RN001.4: O sistema deve validar todos os campos obrigatórios antes de permitir o envio.
RN001.5: Após o envio, o sistema deve gerar um número de protocolo único e exibir uma mensagem de confirmação.
RN001.6: A confirmação deve conter o número do protocolo, data solicitada, e resumo dos dados fornecidos.
RF002: Autenticação de Usuários
O sistema deve ter um mecanismo simples de autenticação para proteger o acesso à área administrativa.
RN002.1: O login deve ser realizado por meio de e-mail e senha. 
Não é necessário implementar o cadastro de usuários
: um usuário administrador pode ser previamente inserido no banco de dados, manualmente ou por seed.
RN002.2: O backend não deve permitir acesso à listagem de agendamentos sem autenticação válida.
RN002.3: O frontend deve redirecionar usuários não autenticados para a tela de login ao tentar acessar rotas protegidas.
RF003: Armazenar e Listar Agendamentos
O sistema deve armazenar as informações dos agendamentos e permitir que 
usuários autenticados
 acessem uma listagem.
RN003.1: Apenas usuários autenticados poderão visualizar a listagem de agendamentos.
RN003.2: A listagem deve exibir pelo menos: nome do cidadão, data da coleta, tipos de materiais e status atual.
RN003.3: A listagem deve permitir filtros por data da coleta e status.
RN003.4: O sistema deve ordenar os agendamentos por data da coleta (mais próximos primeiro) por padrão.
RF004: Detalhamento do Agendamento
O sistema deve permitir a visualização completa das informações de um agendamento específico.
RN004.1: A tela de detalhe deve exibir todas as informações fornecidas pelo cidadão.
RN004.2: A visualização deve estar disponível apenas para usuários autenticados.
RF005: Atualizar Status da Coleta
O sistema deve permitir que usuários autenticados atualizem o status de um agendamento.
RN005.1: Os status possíveis são: “Pendente”, “Agendado”, “Concluído”, “Cancelado”.
RN005.2: O sistema deve registrar a data e hora da última atualização de status.
RN005.3: Ao atualizar o status para “Concluído” ou “Cancelado”, deve haver um campo para justificativa ou observações com preenchimento obrigatório.
RF006: (Diferencial) Gerenciar Tipos de Materiais
O sistema (ou um módulo administrativo) deve permitir o gerenciamento dos tipos de materiais recicláveis.
RN006.1: Deve ser possível cadastrar, editar e remover tipos de materiais.
RN006.2: Cada tipo de material pode ter uma descrição associada e uma categoria (orgânico, seco, eletrônico, etc.).
4. 
Regras de Negócio Gerais Adicionais
RN-G01: Feedback ao Usuário
RN-G01.1: O sistema deve fornecer feedback visual imediato para todas as ações do usuário (envio de formulário, login, filtros, etc.).
RN-G01.2: Mensagens de erro devem ser claras, visíveis e descritivas (ex: "Data inválida", "Preencha todos os campos obrigatórios").
RN-G02: Validação de Campos
RN-G02.1: Sem campos obrigatórios preenchidos o sistema não deve cadastrar em branco de dados.
RN-G02.2: O sistema deve validar o formato do telefone e da data.
RN-G03: Estado da Aplicação
RN-G03.1: O sistema deve manter o estado do formulário em caso de erro na submissão.
